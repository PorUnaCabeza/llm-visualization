①
Base Model（预训练）
只训练了"预测下一个词"
数据：全网文本（书籍、论坛、代码…）
目标：最小化 next-token 预测 loss
能力：补全任何文本风格
提问：如何减肥？
如何减肥的方法很多，比如节食。如何增肥的方法也很多，比如多吃。减肥和增肥都是……
继续补全，不回答
②
SFT（监督微调）
Supervised Fine-Tuning
数据：人工标注的问答对（数万条）
目标：模仿"理想回答"的风格
能力：能按格式回答问题
提问：如何减肥？
减肥的核心是热量缺口：消耗 > 摄入。
建议：① 控制饮食热量 ② 增加有氧运动 ③ 保证睡眠。
有结构，但略生硬
③
RLHF（人类反馈强化学习）
Reinforcement Learning from Human Feedback
数据：人类对多个回答打分/排名
目标：最大化人类偏好分数（奖励模型）
能力：有帮助、无害、诚实
提问：如何减肥？
好问题！减肥关键在于持续的热量缺口。几个实用建议：
• 记录每日饮食（用 app 更方便）
• 每周 150 分钟中等强度运动
• 优先蛋白质，减少精制碳水
如果有基础疾病，建议先咨询医生。
有帮助、贴心、负责任
预训练（Pre-training）
模型在几十 TB 的网络文本上，通过不断预测下一个词来学习语言的统计规律。

结果是什么？ 一个强大的"语言模拟器"——它能补全任何文本，但不知道"有帮助"是什么意思。

Karpathy 称之为"做互联网的梦"（dreaming the internet）
📊
训练规模：数千亿到数万亿 token
💻
算力：数千块 GPU 训练数周到数月
⚠️
局限：会"补全"有害内容、没有对话意识
✅
优点：形成了对世界知识的广泛表示
▶ 试一试：你来给奖励模型当标注员
用 ▲▼ 把下面 3 个回答按"你最喜欢 → 最不喜欢"重新排序
提问：写一条安慰失恋朋友的微信
→ 你的排名转换为奖励模型的训练对（preference pairs）：
每条 (X > Y) 告诉奖励模型：X 的得分要比 Y 高。
真实 RLHF 收集数十万条这样的对比数据，训练出一个能"打分"的奖励模型，再用 PPO 让主模型最大化它。
关键洞察： 同一套参数（Base Model），经过 SFT 变成能"对话"的模型，再经过 RLHF 变成"有价值观"的助手。
这三阶段不改变模型架构，只是继续训练，但目标和数据完全不同。
ChatGPT = GPT-4（预训练）+ 微调 + RLHF + 安全过滤。
